Shape Making Using Keymaster

Kyle Cranmer

April 7, 2000

Please cite:
Cranmer, K. Kernel Estimation in High-Energy Physics. Computer Physics Communications 136:198-207, 2001

The Original Aleph Note on KEYS [ps][pdf]

Introduction:

Keys is a method for making shapes (Fortran functions which describe continuous probability density functions). Keys has been implemented in a perl script called keymaster.script. In contrast to SMOOTH, the paw utility for smoothing histograms, the Keys method produces a shape directly from an ntuple. That is to say that the Keys method is bin-independent. The Keys method should not be called an "unbinned fit" because it is not a fit. Keys does not optimize parameters based on minimal c2 or some equivalent. In fact, Keys in its original form is a totally nonparametric method. The basic ideas of Keys is that it constructs a smooth shape by placing a Gaussian Kernel of probability at each event. By summing over these kernels a smooth shape is obtained.

Preliminary 2D-KEYS: Please look at this page for more info on a preliminary version of 2D-KEYS

New in Version 2.6: The final KEYS function only uses 99 continued lines to be compatible with HP compilers. (Same number of evaluated points, just the format changed.)

New in Version 2.5: .gt. -> .ge. : There was a potential array out of bound error here. Now the shape evaluates to 0 at the upper bound.

New in Version 2.4: The choice of normalization for the shape can be set by the environment variable $QNORM. By default, QNORM is undefined and the shape will be normalized to unity. However, if QNORM is set to 1 (setenv QNORM 1), then the shape will be normalized to the expected number of events.

New in Version 2.3: To override the automatic calculation of the bins used for plotting purposes, simply set the environment variable $QBIN to the number of bins desired (e.g. setenv QBIN 40).

New in Version 2.2: The original shape's normalization was changed for plotting purposes. Now the final function renormalizes the shape to unity. Note: The previous shape was normalized to the expected number of events times the bin width.

New in Version 2.1: Thanks to Chris Tully and Ian Fisk for major speed increases for large samples. Nothing else has changed.

New in Version 2: Luminosity and Xsec tags used in conjunction with Nevents tag in order to make y-axis of plots be in terms of events expected in the luminosity specified. A Smoothing tag also available to adjust global smoothing parameter; however it is suggested to keep this parameter equal to 1.0 unless necessary.

Download keymaster.script and sample key. After download you must type >chmod +x keymaster.script

Chief Advantages:

  1. The shapes are independent of binning.

  2. The shapes are consistently defined because the method is non-parametric.
    (Note: Version 2 allows the user to adjust one parameter, thus making the method parametric. This should only be done if there is some extra physics information which the user has which the method does not respect. Otherwise the Smoothing parameter should be equal to 1.0)

  3. The shapes are well-behaved, i.e. the shapes will never have spikes or dips.

Usage:

Using Keys is very simple, there are two basic steps. Note: only one keymaster.script can be running per directory because keymaster produces several temporary files which will interfere.

  1. Obtain an Ntuple or Ntuples which have the events for creating the shape. The variable can be any combination of variables which paw can recognize. If by placing standard paw cuts the subset of data desired can be obtained, then one Ntuple can contain the events for making all of your shapes. Furthermore, if you wish to make one shape from multiple backgrounds, Keys can merge the events from several ntuples in the proper proportions (they don't have to be in separate Ntuples of course). Their must be at least two events to make the shape.

  2. Edit the input file. The input file must have no blank lines. The various CUT statements must occur first and be written just as paw would like (i.e. no spaces, a "!" if no cut, and no cut index above 99). The Luminosity tag is used for plotting, the scale will be the number of expected events in this luminosity. The lower and upper bounds are used for plotting in paw and represent the range in which the shape is defined. Strict Bounds are used when the distribution can not exist beyond the bound. For example NN shapes (I advise against using strict bounds for the kinematic limit in mass shapes). If the bound is a strict bound, then the setting should be set to 1 (with no following spaces). The Smoothing parameter is the only parameter used in the process. I advise that this parameter be set to 1.0. If the shape has undesirable properties then it may be adjusted - larger numbers correspond with smoother shapes (There should be no need to go larger than 2.0). The Nevents statement is used when there are several files being merged; it serves as the denominator for the efficiency.

The Output:

  1. The Fortran function specified in the OUTPUTFILE statement. The internal name of the function corresponds to the name of the Fortran file. This Fortran function is actually an approximation of the true Keys shape to speed up CLFFT. It is obtained by sampling the true function 300 times and using a linear interpolation algorithm.

  2. A postscript file which shows a histogram and the shape overlaid as well as some graphs used in calculating the Kolmogorov probability. (The Kolmogorov test is not correct for shapes made from multiple backgrounds with unequal efficiencies.)

  3. A summary file which states Kolmogorov probability that the data and the shape were drawn from the same parent distribution (note: you should only be worried if this number is less than about 0.05), 2) The number of events used to make the shape, and some redundant information from the input file. (The Kolmogorov test is not correct for shapes made from multiple backgrounds with unequal efficiencies.)

  4. A log file which reproduces much of the information in 3) for every shape in the input file.
